The Limits page shows your organization's current consumption against plan quotas — how many scans, credits, API keys, members, and repos you've used.

---

## Current usage vs limits

| Resource | What it tracks |
|----------|---------------|
| **Scans** | Scans processed this billing cycle vs. plan maximum |
| **Credits** | Credits consumed vs. plan allocation |
| **API keys** | Active keys vs. plan maximum |
| **Members** | Team members vs. plan maximum |
| **Repositories** | Connected repos vs. plan maximum |

Each resource shows a progress bar with current usage, the limit, and percentage consumed.

## What happens at the limit

| Resource | Behavior |
|----------|----------|
| **Scans** | New scans are rejected with `429 Rate Limited`. Existing data remains accessible. |
| **Credits** | Credit-consuming operations (scans, reviews, fix suggestions) are rejected. Existing data remains accessible. |
| **API keys** | Cannot create new keys. Existing keys continue to work. |
| **Members** | Cannot invite new members. Existing members retain access. |
| **Repositories** | Cannot connect new repos. Existing repos continue to be monitored. |

A warning banner appears when you reach 80% of any limit. At 100%, the affected feature stops and a notification is sent to the Owner and Billing Admin.

---

## Upgrading

Click **Upgrade Plan** on the limits page to increase quotas. Changes take effect immediately — no waiting for the next billing cycle.
