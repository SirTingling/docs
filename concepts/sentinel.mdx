# Sentinel

Sentinel is the inbound protection layer. It validates AI-generated code before it reaches your codebase.

<Note>
Sentinel is in active development. Security scanning and vulnerability detection are available in the dashboard. The standalone API is coming.
</Note>

## The problem

AI generates code fast. That code can contain:

- Security vulnerabilities (SQL injection, XSS, command injection)
- Copyleft-licensed code in your proprietary project
- Deprecated or dangerous APIs
- Patterns that violate your team's standards

You catch these in code review. Sometimes. Sentinel catches them every time.

## What Sentinel does

**Security scanning**
- SQL and NoSQL injection patterns
- Command injection
- Cross-site scripting (XSS)
- Insecure cryptographic usage
- Hardcoded secrets in AI-generated code (yes, AI adds new ones)
- OWASP Top 10 pattern matching

**License compliance**
- Detect GPL/AGPL code in proprietary projects
- Flag license-incompatible dependencies
- Identify code that matches known open source

**Custom rules**
- Patterns to enforce (your team's standards)
- Patterns to forbid (banned APIs, deprecated functions)
- Framework-specific best practices

## How it works

Sentinel runs after the AI responds, before the code reaches your editor:

1. AI generates code
2. The gateway intercepts the response
3. Sentinel scans for vulnerabilities, license issues, and policy violations
4. Clean code passes through. Flagged code is annotated with the issue.

The enforcement mode depends on your policy configuration:
- **Block** — flagged code is rejected
- **Warn** — code passes through with warnings attached
- **Monitor** — code passes through, issue is logged to the audit trail

## In the dashboard

The dashboard shows Sentinel detections under the vulnerability breakdown. You can see:
- Detection type and severity
- Affected code location
- Suggested remediation
- Trend data over time

## Standalone API

The Sentinel API will allow you to validate code outside the gateway flow:

```typescript
const result = await sentinel.validate(aiGeneratedCode, {
  rules: ["security", "license"],
  licensePolicy: "permissive-only",
  customRules: "./takumo-rules.yaml"
});

if (!result.passed) {
  for (const issue of result.issues) {
    console.log(`${issue.severity}: ${issue.message}`);
    console.log(`  Line ${issue.line}: ${issue.snippet}`);
  }
}
```

<Note>
The standalone Sentinel API is in development. Security scanning is available today through the gateway and dashboard.
</Note>
