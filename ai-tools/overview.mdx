# AI Tool Integrations

Takumo works with any AI tool that lets you set a custom API endpoint. Point it at your gateway, and secrets get tokenized before they leave your machine.

## How it works

<Steps>
  <Step title="Your code">
    You write code. It has secrets in it -- API keys, database URLs, tokens.
  </Step>
  <Step title="Gateway tokenizes">
    Your AI tool sends requests through the Takumo gateway. The gateway detects secrets and replaces them with tokens before forwarding to the AI provider.
  </Step>
  <Step title="AI processes tokens">
    The AI model sees tokens, not real values. It writes code around them like normal.
  </Step>
  <Step title="Gateway rehydrates">
    The response comes back through the gateway. Takumo finds the tokens and swaps in your real secrets.
  </Step>
  <Step title="You get working code">
    Your editor receives working code with actual credentials. You never had to do anything.
  </Step>
</Steps>

## No extensions required

<Note>
  This is not a plugin. There is no extension to install, no marketplace to visit, no beta program to join. Takumo is a proxy. You change one URL in your AI tool's settings, and every request goes through your gateway. That's it.
</Note>

## What the gateway does

Every request that passes through the gateway gets the same treatment:

1. **Outbound** -- Scans your code for secrets, replaces them with tokens, forwards to the AI provider
2. **Inbound** -- Scans the AI response for tokens, replaces them with your real secrets, returns to your tool

The AI provider never sees your credentials. Your tool never knows the difference.

```typescript
// What you write
const db = "postgres://admin:supersecret@prod.internal:5432/app";

// What the AI provider receives
const db = "__TAKUMO_v1_CONN_a1b2c3d4__";

// What you get back
const db = "postgres://admin:supersecret@prod.internal:5432/app";
```

Each tool page has specific setup instructions. The pattern is always the same: change the API URL, set your key, done.
