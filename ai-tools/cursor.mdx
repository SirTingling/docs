# Cursor

Cursor supports custom API endpoints. Set your Takumo gateway as the base URL, and every Chat, Composer, and inline edit request gets protected.

## Setup

<Steps>
  <Step title="Get your API key">
    Log into [cloud.takumo.io](https://cloud.takumo.io) and grab your API key from the dashboard. You need this to authenticate with the gateway.
  </Step>
  <Step title="Open Cursor settings">
    Press `Cmd + ,` (macOS) or `Ctrl + ,` (Windows/Linux) to open Settings.
  </Step>
  <Step title="Go to Models">
    Navigate to the **Models** section in Settings.
  </Step>
  <Step title="Set the base URL">
    Find **Override OpenAI Base URL** and set it to your Takumo gateway:

    ```
    https://gateway.takumo.io/v1
    ```

    If you're running a self-hosted gateway, use your own URL instead:

    ```
    https://gateway.your-company.com/v1
    ```
  </Step>
  <Step title="Set your API key">
    Enter your Takumo API key in the API key field. This replaces your OpenAI/Anthropic key -- the gateway handles provider authentication on your behalf.
  </Step>
  <Step title="Test it">
    Open a file that has a secret in it. Send it to Chat or Composer. Check the [Takumo dashboard](https://cloud.takumo.io/dashboard) -- you should see the detection logged.
  </Step>
</Steps>

## Usage

Once configured, everything works the same as before. You don't change how you use Cursor.

### Chat

Attach files with `@`, ask questions, get answers. Secrets in attached files are tokenized before they reach the AI provider.

```
You: @config.ts Add retry logic to the database connection
Cursor: Here's the updated config with exponential backoff...
```

The AI never saw your real database URL. You got working code back with the real URL in place.

### Composer

Multi-file edits work the same way. Every file Cursor sends to the AI goes through the gateway first.

```
You: Refactor authentication across these files
     @auth.ts @middleware.ts @routes/login.ts
```

All secrets across all three files are tokenized. Same secret in different files gets the same token, so the AI understands they're connected.

### Inline edits

Select code, press `Cmd + K` / `Ctrl + K`, describe the change. The selected code is tokenized before it leaves.

```typescript
// You select this line:
const stripe = new Stripe("sk_test_abc123EXAMPLE");

// AI sees:
const stripe = new Stripe("__TAKUMO_v1_KEY_d4e5f6__");

// You get back:
const stripe = new Stripe("sk_test_abc123EXAMPLE");
if (!stripe) throw new Error("Stripe client not initialized");
```

<Note>
  Cursor routes requests through the gateway automatically. You don't need to change how you use Cursor -- just how it connects.
</Note>

## Troubleshooting

<Accordion title="Requests not going through the gateway">
  Verify the base URL is set correctly:

  1. Open Settings (`Cmd/Ctrl + ,`)
  2. Go to **Models**
  3. Check that **Override OpenAI Base URL** is exactly `https://gateway.takumo.io/v1`
  4. Make sure there's no trailing slash or extra path segments
  5. Confirm your API key is set

  If you're on a self-hosted gateway, make sure it's reachable from your machine. Try:

  ```bash
  curl https://gateway.your-company.com/v1/health
  ```
</Accordion>

<Accordion title="Secrets not being detected">
  The gateway detects secrets based on known patterns (AWS keys, database URLs, Stripe keys, etc.) and context analysis (variables named `password`, `secret`, `api_key`).

  If a specific secret isn't being caught:

  1. Check [Supported Patterns](/patterns/supported) to see if the format is covered
  2. Look at the dashboard detection log for what was and wasn't flagged
  3. You can add custom patterns in your organization settings
</Accordion>

<Accordion title="Higher latency than usual">
  The first request in a session has key exchange overhead. Subsequent requests are faster.

  If latency is consistently high:

  1. Check your gateway region -- pick one close to you
  2. Self-hosted gateways eliminate the network hop to Takumo's infrastructure
  3. The dashboard shows per-request latency so you can see where time is spent
</Accordion>
