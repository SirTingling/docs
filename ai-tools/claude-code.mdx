# Claude Code

Claude Code reads environment variables for its API connection. Set two variables and every request goes through Takumo.

## Setup

<Steps>
  <Step title="Get your API key">
    Log into [cloud.takumo.io](https://cloud.takumo.io) and copy your API key from the dashboard.
  </Step>
  <Step title="Set environment variables">
    Two variables. That's the whole setup.

    ```bash
    export ANTHROPIC_BASE_URL="https://gateway.takumo.io/v1"
    export ANTHROPIC_API_KEY="your-takumo-api-key"
    ```
  </Step>
  <Step title="Run Claude Code">
    Use Claude Code like you normally would. Every request routes through the Takumo gateway automatically.

    ```bash
    claude "Refactor the database connection in config.ts"
    ```
  </Step>
  <Step title="Check the dashboard">
    Open the [Takumo dashboard](https://cloud.takumo.io/dashboard). You should see detections logged for any secrets in files Claude Code read.
  </Step>
</Steps>

## Add to your shell profile

Set the variables once so they persist across terminal sessions.

### Bash

```bash
# Add to ~/.bashrc or ~/.bash_profile
export ANTHROPIC_BASE_URL="https://gateway.takumo.io/v1"
export ANTHROPIC_API_KEY="your-takumo-api-key"
```

### Zsh

```bash
# Add to ~/.zshrc
export ANTHROPIC_BASE_URL="https://gateway.takumo.io/v1"
export ANTHROPIC_API_KEY="your-takumo-api-key"
```

### Fish

```fish
# Add to ~/.config/fish/config.fish
set -gx ANTHROPIC_BASE_URL "https://gateway.takumo.io/v1"
set -gx ANTHROPIC_API_KEY "your-takumo-api-key"
```

Reload your shell after adding:

```bash
source ~/.zshrc  # or ~/.bashrc, depending on your shell
```

## Usage

Once the variables are set, Claude Code works exactly the same. You don't change any commands or flags.

```bash
# All of these route through Takumo automatically
claude "Add retry logic to src/db.ts"
claude "Explain what config.ts does"
claude "Find the bug in auth/middleware.ts"
```

Every file Claude Code reads goes through the gateway. Secrets are tokenized on the way out, rehydrated on the way back.

<Note>
  Claude Code doesn't know Takumo is there. It sends requests to `ANTHROPIC_BASE_URL`, which happens to be your gateway. The gateway handles tokenization and rehydration transparently.
</Note>

## Self-hosted gateway

If you're running your own gateway, point to that instead:

```bash
export ANTHROPIC_BASE_URL="https://gateway.your-company.com/v1"
export ANTHROPIC_API_KEY="your-takumo-api-key"
```

Same two variables, different URL.

## Per-project configuration

If you only want Takumo on specific projects, set the variables in a `.envrc` file with [direnv](https://direnv.net):

```bash
# .envrc (in your project root)
export ANTHROPIC_BASE_URL="https://gateway.takumo.io/v1"
export ANTHROPIC_API_KEY="your-takumo-api-key"
```

Then `cd` into the project and the variables are set automatically. Leave the project and they're unset.

## Troubleshooting

<Accordion title="Claude Code ignoring the environment variables">
  Check that the variables are actually set:

  ```bash
  echo $ANTHROPIC_BASE_URL
  echo $ANTHROPIC_API_KEY
  ```

  If they're empty:

  1. Make sure you sourced your shell profile after adding them
  2. If using direnv, run `direnv allow` in the project directory
  3. New terminal windows should pick them up automatically
</Accordion>

<Accordion title="Authentication errors">
  The gateway authenticates you with Takumo, then forwards requests to Anthropic using your organization's provider key. If you're getting auth errors:

  1. Check your Takumo API key is valid (dashboard shows key status)
  2. Make sure your organization has a provider key configured in settings
  3. The `ANTHROPIC_API_KEY` should be your Takumo key, not your Anthropic key
</Accordion>

<Accordion title="Secrets not being detected">
  Run a scan to see what Takumo finds in a specific file:

  ```bash
  takumo-aegis scan src/config.ts
  ```

  If a secret type isn't detected, check [Supported Patterns](/patterns/supported) or add custom patterns in your organization settings.
</Accordion>
