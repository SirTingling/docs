Scan code for security issues using Sentinel rules. Returns detected issues with severity, location, and fix suggestions.

<Note>
This endpoint is served by the **Aegis Shield Gateway** (`gateway.takumo.io`), not the Cloud API. The gateway runs the Sentinel scan engine directly. The same endpoint is also available locally via the [daemon](/concepts/daemon).
</Note>

## POST /scan

Send code to the gateway and get back Sentinel scan results.

<CodeGroup>
```bash curl
curl -X POST https://gateway.takumo.io/scan \
  -H "Authorization: Bearer <api_key>" \
  -H "Content-Type: application/json" \
  -d '{
    "content": "const query = `SELECT * FROM users WHERE id = ${userInput}`;",
    "filename": "handler.ts",
    "language": "typescript"
  }'
```

```typescript TypeScript
const response = await fetch("https://gateway.takumo.io/scan", {
  method: "POST",
  headers: {
    "Authorization": "Bearer <api_key>",
    "Content-Type": "application/json"
  },
  body: JSON.stringify({
    content: 'const query = `SELECT * FROM users WHERE id = ${userInput}`;',
    filename: "handler.ts",
    language: "typescript"
  })
});
const result = await response.json();
```

```python Python
import requests

result = requests.post(
    "https://gateway.takumo.io/scan",
    headers={"Authorization": "Bearer <api_key>"},
    json={
        "content": 'const query = `SELECT * FROM users WHERE id = ${userInput}`;',
        "filename": "handler.ts",
        "language": "typescript"
    }
).json()
```
</CodeGroup>

### Request body

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `content` | string | Yes | Code to scan |
| `filename` | string | No | File name (helps with language detection) |
| `language` | string | No | Language identifier |

### Response

```json
{
  "scan_id": "scan_abc123",
  "passed": false,
  "issues": [
    {
      "rule_id": "sql-injection",
      "severity": "critical",
      "message": "Unsanitized user input in SQL query",
      "line": 1,
      "column": 43,
      "fix_suggestion": "Use parameterized queries instead of string interpolation"
    }
  ],
  "engine_duration_ms": 12
}
```

| Field | Type | Description |
|-------|------|-------------|
| `scan_id` | string | Unique scan identifier |
| `passed` | boolean | `true` if no issues found |
| `issues` | array | List of detected issues |
| `engine_duration_ms` | number | Scan engine time in milliseconds |

### Issue object

| Field | Type | Description |
|-------|------|-------------|
| `rule_id` | string | Rule that triggered (e.g., `sql-injection`, `xss`, `SEC-001`) |
| `severity` | string | `info`, `warning`, `error`, or `critical` |
| `message` | string | Human-readable description |
| `line` | number | Line number (1-indexed) |
| `column` | number | Column number (1-indexed) |
| `fix_suggestion` | string | Suggested fix |

## Local daemon

The daemon exposes the same endpoint locally on port `19532`. No authentication is required for local requests.

```bash
curl -X POST http://localhost:19532/scan \
  -H "Content-Type: application/json" \
  -d '{"content": "eval(userInput)", "filename": "app.js"}'
```

The daemon runs the Sentinel scan engine offline -- it does not require network access to the gateway. Scan results from both the gateway and daemon use the same response format.
