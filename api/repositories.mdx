List repositories connected to your organization with their security posture, compliance scores, and risk levels.

<Note>Requires Enterprise plan and `scan:read` scope on the API key.</Note>

## GET /repositories

<CodeGroup>
```bash curl
curl "https://cloud.takumo.io/api/v1/repositories?riskLevel=high" \
  -H "Authorization: Bearer tk_live_your_api_key"
```

```typescript TypeScript
const response = await fetch(
  "https://cloud.takumo.io/api/v1/repositories?riskLevel=high",
  { headers: { "Authorization": "Bearer tk_live_your_api_key" } }
);
const result = await response.json();
```
</CodeGroup>

### Query parameters

| Parameter | Type | Required | Description |
|-----------|------|----------|-------------|
| `criticality` | string | No | Filter by criticality level |
| `riskLevel` | string | No | Filter by risk level |
| `visibility` | string | No | Filter by visibility (e.g. `public`, `private`) |
| `search` | string | No | Search by repository name, full name, or language (case-insensitive) |
| `cursor` | string | No | Pagination cursor (pass `nextCursor` from a previous response) |
| `limit` | number | No | Results per page (default 50, max 100) |

## GET /repositories/:owner/:repo

Get detailed information for a specific repository. The path uses the `owner/repo` format.

<CodeGroup>
```bash curl
curl "https://cloud.takumo.io/api/v1/repositories/org/api-service" \
  -H "Authorization: Bearer tk_live_your_api_key"
```

```typescript TypeScript
const response = await fetch(
  "https://cloud.takumo.io/api/v1/repositories/org/api-service",
  { headers: { "Authorization": "Bearer tk_live_your_api_key" } }
);
const result = await response.json();
```
</CodeGroup>

### Repository object (list)

| Field | Type | Description |
|-------|------|-------------|
| `id` | string | Unique repository ID |
| `fullName` | string | Full repository name (`owner/repo`) |
| `owner` | string | Repository owner |
| `name` | string | Repository name |
| `defaultBranch` | string | Default branch name |
| `visibility` | string | `public`, `private`, or `internal` |
| `language` | string \| null | Primary language |
| `topics` | string[] | Repository topics |
| `archived` | boolean | Whether the repository is archived |
| `disabled` | boolean | Whether the repository is disabled |
| `criticality` | string | Criticality level |
| `criticalityReason` | string \| null | Why this criticality was assigned |
| `securityPosture` | object | Security configuration summary |
| `securityPosture.branchProtectionEnabled` | boolean | Branch protection rules active |
| `securityPosture.branchProtectionUnavailable` | boolean | Whether branch protection data could not be fetched |
| `securityPosture.requiredReviewers` | number | Minimum required PR reviewers |
| `securityPosture.secretScanningEnabled` | boolean | GitHub secret scanning active |
| `securityPosture.dependabotEnabled` | boolean | Dependabot alerts active |
| `securityPosture.codeScanningEnabled` | boolean | Code scanning (CodeQL etc.) active |
| `complianceScore` | number | Compliance score (0-100) |
| `riskLevel` | string | Computed risk level |
| `openIssueCount` | number | Unresolved security issues |
| `openViolationCount` | number | Unresolved policy violations |
| `lastPushedAt` | string \| null | Last push timestamp |
| `lastSyncAt` | string \| null | Last sync with source provider |
| `createdAt` | string | Record creation timestamp |
| `updatedAt` | string | Last update timestamp |

### Additional fields (detail only)

| Field | Type | Description |
|-------|------|-------------|
| `sbom` | object \| null | SBOM summary, if available |
| `sbom.summary` | object | Parsed SBOM data (package counts, ecosystems, licenses) |
| `sbom.fetchedAt` | string \| null | When the SBOM was last fetched |
| `codeOwners` | array \| null | Parsed CODEOWNERS entries |
| `codeOwners[].pattern` | string | File pattern |
| `codeOwners[].owners` | string[] | Owners for matching files |
